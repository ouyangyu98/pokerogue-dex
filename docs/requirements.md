# PokeRogue 本地图鉴工具 PRD（一期）

**版本：** 1.3
**日期：** 2026-06-19

---

## 1. 产品定位

一期先做本地图鉴工具，目标是把 PokeRogue 的精灵资料、地区遭遇资料、进化形态资料、技能获取顺序整理到一个本地页面中，方便玩家快速查询。

本期交付重点是**数据真实、字段完整、查询可用**。项目核心验收对象是"真实全量数据 + 可查询筛选"，不是 UI Demo。任何仅使用 mock 数据、少量手写数据或通用宝可梦数据库拼出的版本，只能视为原型，不视为一期交付。

本项目只考虑桌面端浏览器使用，不考虑移动端适配。页面按桌面端设计，建议适配 1440px 及以上宽度；低于该宽度允许横向滚动。

## 3. 数据来源策略

一期必须优先完成数据导入与数据结构设计，不允许将数据硬编码在页面逻辑中。

考虑到 PokeRogue Wiki 可能被 Cloudflare 安全验证拦截，国内 AI 编程工具不得依赖 Wiki 网页抓取作为主数据源。Wiki 仅作为人工核验与字段补充来源，不作为自动化数据导入主路径。

### 数据来源优先级
1. 官方 GitHub 源码数据
2. 用户手动下载的官方源码压缩包
3. 成熟第三方图鉴项目的数据结构与转换脚本
4. PokeRogue Wiki 人工校验
5. 人工补录

### 本地源码包方案
如果 AI 工具无法稳定访问 GitHub 或 Wiki，应采用"本地源码包方案"：由用户手动下载 PokeRogue 官方源码压缩包，放入项目的 `source/pokerogue/` 目录，AI 只读取本地源码文件并转换为本项目所需 JSON。

若网络安装依赖失败，需说明失败原因并切换 npm 镜像或让用户提供离线依赖方案，不得因此改成纯 HTML 或删减功能。

## 6. 数据建模要求

- 数据层和展示层必须分离
- 数据文件不得混入渲染逻辑，页面逻辑不得依赖示例数据的固定顺序或固定数量
- 官方源码中的 TypeScript 数据可能存在枚举、常量引用、数组索引、分散配置等结构
- 数据转换脚本应允许复用项目 TypeScript/Node 运行环境，优先通过导入源码模块、AST 解析或明确映射表转换
- **不得依赖脆弱的字符串正则硬抠核心字段**

### 中文映射要求
- 所有核心实体需支持中文名、英文名、内部 ID 查询
- **中文展示为默认展示语言**，英文名作为辅助字段
- 中文名缺失时才回退英文名
- 中文映射范围包括：精灵、形态、属性、技能、特性、被动、地区、稀有度、技能来源类型
- 未映射字段允许英文回退，但需在数据统计中显示中文映射覆盖率
- 中文映射覆盖率需按实体类型分别统计：精灵、形态、属性、技能、特性、地区、稀有度、技能来源类型

### 数据规范
- 缺失字段统一使用 `null` 或空数组，不允许同一字段混用空字符串、`undefined`、`"-"`
- 筛选时空数组按"不具备该能力"处理
- 形态、进化、技能、特性、遭遇记录必须使用数组对象结构，不得用纯文本拼接
- 概率字段统一存储为百分比数值，例如 `0.63` 表示 0.63%，不是 0.0063
- 前端展示时统一追加 `%`
- 核心 TypeScript 数据结构禁止使用 `any` 兜底

## 11. 地区导航

### 功能说明
- SVG 中文地图展示所有生态区（35 个节点）
- 节点按类型着色（水系蓝色、火系红色、草系绿色等）
- 连线表示源码中定义的可通行路径（单向）
- 支持通过下拉框或点击地图节点选择起点/终点
- 使用 BFS 算法计算最短路线
- 路线结果展示每步的通行概率

### 概率规则
- 源码中 `biomeLinks` 无权重 → 100% 通行
- 源码中 `biomeLinks` 有权重 `n` → `1/n` 概率通行（如 weight=2 表示 50%，weight=3 表示 33%）
- 严格遵循源码单向连接，不假设双向通行

### 交互设计
- 左侧：SVG 地图（可点击节点快速选择）
- 右侧：起点/终点选择器 + 计算按钮 + 路线结果
- 路线高亮：地图上的路线节点和连线高亮显示
- 起点标记绿色，终点标记红色

## 17. 配队分析器（Team Builder）

### 定位
- **独立页面 / 独立导航入口**，不与精灵列表、属性克制、地区导航混排

- 核心回答：属性覆盖够全吗？有没有共同弱点？缺什么职能？对面某属性强势时整体抗性如何？

### 核心功能模块

#### 17.1 队伍配置（输入层）
- **6 个精灵槽位**，支持：
  - 点击空位 → 弹出**大弹窗精灵选择器**（复用列表页搜索/筛选/表格逻辑，单选模式）
  - 已选精灵显示缩略图 + 中文名 + 属性 badge
  - 点击已选精灵可**切换形态**（复用详情页 `selectedFormIndex` 逻辑）
  - 点击 × 移除
- **快捷操作**：
  - "清空队伍"
  - "保存队伍" → 命名后存到 `localStorage`
  - "加载队伍" → 从已保存列表恢复

#### 17.2 属性覆盖分析
- 基于队伍 6 只精灵的**本系属性** + **全技能池属性**（非仅配招推荐的 4 格）计算
- 指标：本系属性分布、技能属性覆盖、未覆盖属性（盲区）、重复度（某属性被多少只精灵共享）

#### 17.3 抗性总览（防御面）
- 基于 6 只精灵的实际属性组合，计算队伍整体防御矩阵
- 指标：弱点数 / 抗性数 / 免疫数（按 18 种攻击属性统计）
- **危险属性预警**：超过 3 只精灵共同弱点的属性标红

#### 17.4 职能分布
- 基于种族值和全技能池，自动给每只精灵打标签：
  - 物攻手（物攻 ≥ 100 且有物理输出技能）
  - 特攻手（特攻 ≥ 100 且有特殊输出技能）
  - 高速（速度 ≥ 100）
  - 耐久型（HP+防御+特防 ≥ 280）
  - 强化手 / 回复手 / 控场手 / 转场手（基于技能池判定）
- 统计各标签在队伍中的数量分布

#### 17.5 缺口建议（决策输出）
- 基于上述分析生成**方向性文字建议**，不做具体精灵推荐
- 示例："队伍有 3 只精灵弱格斗，建议补充一只格斗抗性精灵"、"缺少高速精灵，当前最快为 85 速"

### 与现有模块边界
- 不并入精灵列表页、属性克制页、地区导航页
- 不替代精灵详情页的配招推荐（单精灵 vs 队伍层面互补）
- 属性克制计算复用 `typeMatchups.ts`，不另写一套

### 首版 MVP 范围
**必做**：
- 6 槽位队伍配置 + 大弹窗精灵选择器
- 属性覆盖分析（本系 + 全技能池属性）
- 抗性总览（弱点/抗性/免疫统计 + 危险预警）
- 职能分布标签
- 缺口建议文字
- 保存/加载队伍到 `localStorage`

**后续可选**：
- 拖拽排序
- 多支队伍管理（命名/删除/导出）
- 导出队伍为文本/图片
